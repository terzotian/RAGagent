## 二、逐項對照評估

### 1. 實用性（權重 40%）——評分：30/40

**優點：**

- **痛點分析紮實到位（第3章）：** 報告從學生、教職員工、行政人員三個維度系統梳理了信息檢索的痛點（文件分散、檢索低效、理解困難、諮詢渠道受限等），論述有針對性且貼近實際場景，令評審能感受到項目的現實需求基礎。
- **應用價值明確（第11章）：** 預期效益覆蓋學生（信息檢索時間降低70%+）、教師（減少重複答疑）、行政（數據驅動決策），三方受益的場景描述具體且可信。
- **轉化前景清晰：** 報告提出了12章「Future」（雖內容缺失），以及10章的維護運營計劃，包括知識庫定期更新、模型優化、專門運營團隊等，展示了長期運營的意圖。
- **實際代碼已實現核心功能：** 經核查代碼庫，系統已實現端到端的 Streaming Q&A、混合檢索（pgvector + TF-IDF + RRF）、文件管理、會話管理、用戶反饋等功能，證明項目不僅停留在紙面。

**不足與改進建議：**

- **第12章「Future」為空白：** 這是嚴重的結構性缺失。未來展望是評估轉化前景的關鍵依據，空白章節直接削弱了實用性的說服力。**建議：** 補充具體的未來規劃，如多語言支持路線圖、與其他校園系統（SIS、LMS）的集成計劃、AI能力升級路徑等。
- **缺乏用戶驗證數據：** 報告提出"準確率95%以上"、"檢索時間降低70%以上"等量化目標，但未提供任何實際測試數據、用戶試用反饋或A/B對比實驗結果。**建議：** 補充小規模用戶測試（pilot test）的結果，哪怕是5-10人的試用數據也能大幅增強說服力。
- **社會效益論述偏局部：** 實用性分析主要聚焦嶺南大學內部，未充分論述對高教行業的示範意義或更廣泛的社會效益（如其他院校的可推廣性）。**建議：** 增加一段關於模式可推廣性的論述。
- **預算估算缺乏依據：** 65-80萬港幣的預算粗略分項，但未說明估算方法和市場參考價格。

### 2. 科學性（權重 20%）——評分：14/20

**優點：**

- **技術框架選型合理（第5.1章）：** AgenticRAG 框架結合了 Agent 編排、語義檢索和生成式 AI，理論上是當前 RAG 系統的主流且先進的方案。報告正確識別了純關鍵字搜索的侷限性，並選用了語義檢索作為替代方案。
- **技術棧選擇有據可查：** React + TypeScript + Vite（前端）、FastAPI（後端）、PostgreSQL + pgvector（向量數據庫）、Google Vertex AI Gemini + Ollama fallback（LLM），這些組件在業界有成熟案例，搭配合理。
- **混合檢索策略科學：** 採用 pgvector 向量檢索 + TF-IDF 文本檢索 + RRF（Reciprocal Rank Fusion）融合排序，這是學術界和工業界驗證過的有效檢索策略，科學性較強。
- **架構文檔質量高：** 實際代碼庫中的 `ARCHITECTURE.md` 詳細記錄了系統設計決策，並標註"code is source of truth"，體現了良好的工程科學態度。

**不足與改進建議：**

- **缺乏理論框架引用：** 報告未引用任何學術論文來支撐其技術選擇。例如，RRF 方法源自 Cormack et al. (2009)，RAG 框架源自 Lewis et al. (2020)，Agent 編排有多篇相關研究——這些引用的缺失使得科學性論證顯得薄弱。**建議：** 在技術方案描述中加入關鍵學術引用，如 RAG 原始論文、RRF 排序論文、Embedding 模型選擇依據等。
- **研究方法描述不充分：** 報告未說明如何評估系統的準確率（是用什麼測試集？什麼指標？Precision/Recall/F1？），也未解釋混合檢索中 TF-IDF 和向量檢索的權重如何確定。**建議：** 補充評估方法論，包括測試數據集的構建方法、評估指標的選擇依據。
- **Embedding 維度選擇未論證：** 代碼中固定使用768維嵌入（`Vector(768)`），但未在報告中解釋為何選擇此維度，是否與 `text-embedding-004` 模型匹配，是否做過維度對比實驗。
- **可行性分析中的經濟可行性缺乏量化支撐：** 聲稱"投資回報週期短"但未給出ROI計算或參考案例。

### 3. 創新性（權重 20%）——評分：13/20

**優點：**

- **選題切合實際需求：** 將 AgenticRAG 應用於大學智能諮詢是一個具有現實意義的選題，將前沿AI技術與高校服務痛點結合，方向正確且有一定創新性。
- **Agent 編排設計有亮點：** 系統採用了"Master Planner + Query Rewrite + Hybrid Retrieval + RRF"的 Agent 鏈式處理流程，不是簡單的"問題→檢索→回答"單步流程，而是加入了查詢改寫、搜索範圍規劃等智能環節，體現了一定的設計創新。
- **角色化知識庫管理：** 系統支持按角色（學生、教師、管理員）和知識類型（政策文件、課程資料、作業文件）進行差異化管理和訪問控制，這種細粒度的知識管理策略在同類項目中不常見。
- **多模態文件支持：** 系統支持 PDF、DOCX、Markdown 等多種文件格式的解析和索引，具備一定的靈活性。

**不足與改進建議：**

- **核心技術框架非原創：** AgenticRAG、混合檢索、RRF 等均為已有成熟方案的組合應用，報告未充分強調自身的獨特創新點。**建議：** 明確列出本項目相對於現有開源 RAG 框架（如 LangChain、LlamaIndex）的差異化設計和改進。
- **缺乏與同類系統的對比：** 報告提及"一些知名大學已推出智能客服系統"，但未具體列舉和對比，也未說明本系統在何處超越了這些先例。**建議：** 增加一個對比分析章節，列出2-3個同類系統，表格化對比功能和技術方案的差異。
- **未展示創新實驗結果：** 未提供任何定量實驗來證明所提出方案（如混合檢索 vs 純向量檢索、有Agent編排 vs 無Agent編排）的優越性。**建議：** 設計消融實驗（ablation study），對比不同組件對系統性能的貢獻。
- **Future 章節缺失意味著前瞻性創新視野未展現。**

- **階段化實施計劃（第7.2-7.3章）：** 四個階段、11-14週的計劃有清晰的里程碑和交付物。

## 四、綜合評估報告

### 整體評價

本報告圍繞嶺南大學智能諮詢系統展開，選題具有明確的現實需求和應用價值，技術方案（AgenticRAG + 混合檢索 + Vertex AI/Ollama）在方向上是正確且合理的。報告結構完整、語言流暢、團隊分工明確，並且有實際可運行的代碼支撐（非純概念性方案），這些都是顯著的優勢。

然而，報告在多個關鍵環節存在明顯的不足，影響了整體評分：

### 三大主要優勢

1. **選題務實，痛點精準：** 從三類用戶角度深入分析了信息檢索的實際困難，需求論證有力。
2. **技術路線成熟可靠：** 技術棧選型合理，AgenticRAG 框架結合混合檢索的方案具有工程可行性，且已有代碼實現。
3. **報告結構邏輯清晰：** 從背景到結論的敘事線完整，章節安排合理，專業術語使用準確。

### 三大關鍵改進領域

1. **嚴謹性亟待加強（最關鍵）：** 缺少學術文獻引用、實驗數據和性能基準測試，這使得報告的科學嚴謹性和可信度大打折扣。這是所有改進點中最優先需要解決的問題。
2. **報告完整性需要修補：** 第12章空白、缺乏架構圖/流程圖/界面截圖、文檔與代碼不同步等問題嚴重影響了報告的專業形象和完整度。
3. **創新性差異化不足：** 需要更清晰地闡述本項目相對於現有方案的獨特創新點，並通過對比實驗或消融實驗來定量證明方案的優越性。
